Responsive Images Demo

Resize the browser to watch different image sources load. Open DevTools → Network to observe which file is requested at each width.

1) img with srcset and sizes

The browser selects the most appropriate file based on viewport size and pixel density. We provide width descriptors (w) and let the browser choose.

  • Small screens (≤600px): downloads up to ~100vw (the full viewport width).
  • Medium screens (≤900px): image displayed at ~50vw; a smaller file is chosen.
  • Large screens: fixed layout width of ~480px suggested by the last value.
  • Lazy loaded via loading="lazy" to defer offscreen images.

2) picture for art direction and formats (WebP/AVIF)

Serve modern formats when supported and fall back to JPEG/PNG. media queries can swap crops for different viewports.

  • Modern formats: AVIF/WebP provide smaller files at similar quality.
  • Fallback: img inside picture ensures compatibility.
  • Art direction: Add media on sources to swap crops by viewport.

3) Density switching with x descriptors

Provide higher-density images for Retina/HiDPI displays using 1x, 2x descriptors.
